# Submission Materials

This directory contains all materials related to Android and iOS app store submission for `acoustic_platform` and `species_monitoring_platform`.

## Files

| File | Purpose |
|------|---------|
| `01_android_ios_submission_checklist.md` | Comprehensive checklist of all items required for Android (Google Play) and iOS (App Store) submission, including permissions, privacy, store assets, and pre-submission technical checks. |
| `02_submission_material_template.md` | Fill-in template for preparing the actual submission package — app info, version notes, privacy declarations, permission justifications, test accounts, and store listing copy. |
| `03_missing_required_business_inputs.md` | Tracks business-side blocking items that must be provided before submission (test accounts, privacy policy URLs, store assets, permission justifications, iOS-specific requirements). |
| `04_release_execution_runbook.md` | Step-by-step D-Day execution guide: pre-checks, environment configuration, frontend builds, Android release packaging, rollback procedures, and go/no-go criteria. |
| `05_go_live_decision_report.md` | Current-round launch decision report summarizing technical verification results, available assets, and remaining actions before submission. |
| `refactor_rebuild_progress_2026-04-23.md` | Progress log for the April 2026 platform refactoring effort (map, tracking, and recording features). |

## Workflow

1. Start with `01` to understand all required materials.
2. Use `02` as the working document to fill in submission details.
3. Track outstanding business inputs in `03`.
4. On release day, follow `04` step-by-step.
5. Use `05` for the final go/no-go decision.
