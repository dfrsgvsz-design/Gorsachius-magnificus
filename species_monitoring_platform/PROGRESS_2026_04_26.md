## 三、修复清单（按文件）

### 1. `frontend/src/lib/surveyOffline.js` — 离线地图缓存统一（**B1**）

**根因**: 三套不同的瓦片缓存桶 — service worker 用 `bird-tile-cache-v4`，`tileCache.js` 用 `biodiversity-survey-tiles-v1`，`prefetchMapTiles` 用 `bird-platform-field-tiles-${cacheKey}`。用户预下载的瓦片永远不会被服务工作者命中，离线场景下地图空白。

**修复**:
- 替换 `TILE_CACHE_PREFIX = "bird-platform-field-tiles"` 为 `SHARED_TILE_CACHE_NAME = "bird-tile-cache-v4"`，与 `service-worker.js` 中的 `TILE_CACHE` 完全一致
- `prefetchMapTiles` 改为写入共享 cache，service worker `staleWhileRevalidateTile` 离线时直接命中
- 区分代理 tile 与跨域 tile：代理走 CORS 请求保留干净 Response，跨域用 `no-cors` 但不持久化失败响应（`response.status === 0` opaque 视为成功，否则计入 `failed`）
- `cacheKey` 参数保留以兼容调用方但不再影响桶名

### 2. `frontend/src/constants.js` — 默认 Tab + NAV 死代码清理（**B2**, **B4**）

**修复**:
- `DEFAULT_TAB_ID`：`"dashboard"` → `"fieldops"`，野外团队首屏直达调查工作台
- 删除 `PRIMARY_NAV_TAB_IDS`、`DEFERRED_TAB_IDS`、`VISIBLE_NAV_GROUP_IDS`、`DEFERRED_NAV_GROUP_IDS` 共 4 个常量
  - `VISIBLE_NAV_GROUP_IDS` 引用 `"analysis"`/`"monitoring"`/`"advanced"`，但 `NAV_GROUPS` 实际定义的是 `"data"`/`"tools"` —— ID 集合完全不交叉，是历史重构遗物
  - 全工作区 grep 无任何文件 import 这些常量

### 3. `frontend/src/main.jsx` — Service Worker 默认在 PROD 启用（**B3**）

**根因**: README 宣称 offline-first，但 SW 仅在 `import.meta.env.PROD && import.meta.env.VITE_ENABLE_SW === 'true'` 时注册。生产构建若未显式设置 `VITE_ENABLE_SW=true`，离线能力静默失效。

**修复**: 反转语义为 opt-out（`VITE_ENABLE_SW=false` 时关闭）。所有生产构建默认启用 SW，匹配发布契约。

### 4. `frontend/src/components/tabs/FieldOpsTab.jsx` — 使用 useGeolocation hook + 预加载回退（**B5**, **B6**）

**B5 修复**:
- 引入 `useGeolocation` hook
- 删除 `const [currentPosition, setCurrentPosition] = useState(null)` 显式声明 + 第 270-292 行手写一次性 GPS 请求 useEffect（约 23 行重复代码）
- 改为 `const { currentPosition, setCurrentPosition } = useGeolocation()`，在 `useTrackRecording` 调用前完成定位

**B6 修复**:
- `preloadTilesOnlineAware()` 不再要求 `currentProjectId` 才能开始
- 即使无项目也允许预下载瓦片到共享缓存（首次设备启动场景）
- 仅当 project_id 存在才注册 server-side map_package
- 增加 `failed_tiles` 上报字段

### 5. `frontend/src/components/fieldops/MapToolsPanel.jsx` — 预加载按钮不再依赖项目（**B6 续**）

**修复**: 解除"预下载离线地图"按钮的 `!currentProjectId` 禁用条件；导入路线按钮保留项目要求（路线必须挂载到项目）。新增清晰的 cursor 状态视觉反馈。

### 6. `frontend/src/hooks/useTrackRecording.js` — 清理未使用导入（**B7**）

**修复**: 删除未使用的 `useCallback` 导入。
