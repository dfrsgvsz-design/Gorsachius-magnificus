# 多角色并行 Git 操作规约（A/B/C/D 共用工作区）

> 生效日期：2026-06-11 ｜ 起草：A（后端 & 架构）｜ 适用：本仓库所有角色会话
>
> 背景：本仓库由多个并行会话（A 后端 / B 前端 & 端到端 / C QA / D 算法）**共用同一个工作目录
> `F:\Gorsachius magnificus` 和同一个 git 索引**。任何一个会话执行 `git add` / `git commit` /
> `git checkout`，其他会话都会立刻受影响。2026-06-11 晚已发生两起互踩事故（见附录），故立此规约。

## 一、分支属主表（只许属主提交）

| 分支 | 属主 | 用途 |
|---|---|---|
| `main` | 共有 | 主干。只接受合并与跨角色治理类提交（文档/规约），提交前需在对话里知会 |
| `batch/a-batch7` | A | 后端加固 / alembic / release gate |
| `batch/b-batch8` | B | 前端 field-ops / e2e 测试 |
| `qa/ticket-c-batch5` | C | QA 工单与质量报告 |
| `batch/d-*`（未来） | D | 算法 / 数据管线 |

跨属主操作（动别人分支、动共享文件）必须先在对话里打招呼，得到属主确认后再动。

## 二、提交前「三查」（每次 commit 前必做）

```powershell
# 1) 查分支：输出必须是你自己的工单分支
git branch --show-current

# 2) 查暂存：清单必须 100% 是你自己的文件
git diff --cached --name-only

# 3) 查工作区：了解哪些改动是别人的，绝不顺手带走
git status --porcelain
```

- 分支不对 → 先协调切换（见第四条），**严禁直接在别人分支上提交**。
- 暂存里混入别人的文件 → `git reset HEAD -- <路径>` 移出后再提交。

## 三、加文件与提交的硬规矩

1. **只用显式路径**：`git add <明确的文件路径>`。
   **禁用**：`git add -A`、`git add .`、`git commit -a`（一把梭必带走别人的东西）。
2. 提交后立即自查：`git show --stat HEAD`，确认文件清单只含自己的；带错了就按第六条 SOP 处理。
3. 提交消息内不要使用英文双引号 `"`（PowerShell 传参会被截断，已踩过坑）；
   长消息一律写入临时文件后 `git commit -F <文件>`。
4. 身份统一用仓库惯例 `Cursor Agent <cursor-agent@local>`（用 `git -c` 临时参数，不改全局配置）。

## 四、切分支纪律（HEAD 是公共资源）

- 当前检出分支是**所有会话共享的**。切分支前先问一句：现在有没有别的角色正在提交？
- 切走之前确认工作区未提交文件与目标分支兼容（git 会拦截冲突，但别赌）。
- 自己工作告一段落、要长时间挂起时，**不要把 HEAD 停在别人的分支上**。

## 五、共享文件的修改规则

`.gitignore`、`shared/backend/**`、根目录配置等多人都会动的文件：

1. 谁要动，先在对话里声明一句（一行即可）。
2. 接受他人版本时用「byte-identical accept」法：直接取对方 commit 里的文件原样提交，
   并比对 blob 指纹（`git rev-parse <commit>:<path>`）确认零漂移——工单 #7 即按此闭环。

## 六、事故处置 SOP

| 事故 | 处置 |
|---|---|
| 提交带错了别人的文件（未推送） | `git reset --soft HEAD~1` → 移出不属于自己的暂存 → 用路径限定重新提交（`git commit -F <msg> -- <自己的路径...>`） |
| 提交落在了别人的分支上（未推送） | 通知属主暂停 → 用临时 worktree 在正确分支上 cherry-pick 移植 → 经双方确认后把错放分支的指针移回（纯 ref 移动，不碰工作区） |
| 需要在非检出分支上落提交 | 用底层命令（`commit-tree` + `update-ref`）直接写目标分支，全程不切分支、不碰共享工作区 |
| 任何已推送的历史 | 一律不改写，只做前滚（revert / 新提交） |
