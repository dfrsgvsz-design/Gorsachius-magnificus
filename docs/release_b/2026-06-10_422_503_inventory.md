## 1. Executive Summary

| Symptom | Original Hypothesis (Work Order) | Verified Root Cause (Repo Audit) | Status |
|---|---|---|---|
| `GET /api/surveys/taxonomy/search` → 422 missing `submodule` | Pydantic model missing `submodule` field | **Field present on both platforms** (species `routes/taxonomy.py:51`, acoustic `routes/taxonomy.py:33`). Stale ticket — most likely an old client deployed before the `submodule` field was added. No code change needed. | ✅ Verified clean |
| `POST /api/surveys/projects` → 503 Survey store unavailable | Module-level lazy loading; stores not initialized | (1) `acoustic_platform/backend/main.py` was **missing** the `sys.modules.setdefault("main", sys.modules[__name__])` alias that the species platform already had (line 25). When uvicorn loaded the app as `backend.main`, every `import main as _m` in `routes/*.py` created a *second* module object whose `survey_store` stayed `None`. (2) Both platforms used the deprecated `@app.on_event("startup")` decorator with no fail-fast assertion: if any store init silently failed, every dependent endpoint would 503 for the lifetime of the process. | ✅ Fixed |
| Demo-mode runtime warning at startup | Hardcoded paths in `runtime_paths.py` | Paths were already overridable via `BIRD_PLATFORM_*` env vars, but (a) `FRONTEND_DIST_DIR` was hardcoded, (b) preferred B-stage env var names (`SURVEY_DATA_DIR`, `CHECKPOINTS_DIR`, `FRONTEND_DIST_DIR`) were not honored, (c) no `release_gate` step verified the contract. | ✅ Fixed |

## 3. 422 Inventory (FastAPI Pydantic validation on required fields)

422 in this project comes almost exclusively from **FastAPI auto-validation** of Pydantic request bodies — not from explicit `raise HTTPException(status_code=422)`. The endpoints below will 422 when the listed *required* fields (no default) are missing or wrong-typed.

| Endpoint | Method | Required body fields | Schema |
|---|---|---|---|
| `/api/search-xc` | POST | `species` | `XCSearchRequest` |
| `/api/config/xc-api-key` (set) | POST | `key` | `APIKeyRequest` |
| `/api/surveys` | POST | `site_name` | `SurveyCreateRequest` |
| `/api/compare-sites` | POST | `sites` (`List[SiteDetections]`, items need `site_name`, `species`) | `CompareSitesRequest` |
| `/api/devices/register` | POST | `name` | `DeviceRegisterRequest` |
| `/api/batch/scan` | POST | `directory` | `BatchScanRequest` |
| `/api/detections/{id}/verify` | POST | `detection_id`, `status` | `VerifyRequest` |
| `/api/detections/verify-batch` | POST | `detection_ids`, `status` | `BatchVerifyRequest` |
| `/api/occupancy/prepare` | POST | `species` | `OccupancyRequest` |
| `/api/surveys/sync/push` (per operation) | POST | `entity_type` per op | `SyncPushRequest`→`SyncOperationRequest` |

These are **intentional input validation** — not bugs. Frontend MUST send these required fields or accept the 422.

## 4. Fixes Applied in This Batch

| Change | Files Touched | Effect |
|---|---|---|
| Added `sys.modules.setdefault("main", sys.modules[__name__])` alias to acoustic platform | `acoustic_platform/backend/main.py` (top of file) | Eliminates the "two main module objects" 503 cascade when uvicorn loads as `backend.main`. Matches the existing species fix at `species_monitoring_platform/backend/main.py:25`. |
| Migrated `@app.on_event("startup")` → modern `lifespan` context manager with **fail-fast store validation** | both `main.py` | Required stores (`survey_store`, `detection_store`, `taxonomy_catalog`) are now asserted non-`None` post-startup. A missing store now raises `RuntimeError` at boot instead of silently returning 503 on every dependent request. |
| Added new env-var aliases `SURVEY_DATA_DIR`, `CHECKPOINTS_DIR`, `FRONTEND_DIST_DIR` (preferred); kept `BIRD_PLATFORM_*` as legacy fallback | 3× `runtime_paths.py` (species, acoustic, `shared/backend/utils`) | Externalizes the previously-hardcoded `frontend/dist` path and gives the B-stage release pipeline the env var names it expects. Reports `*_externalized` flags in `describe_runtime_paths()` for monitoring. |
| Created `shared/backend/utils/data/platform_config.json` as the production default | new file | Both platforms' `platform_config.py` already import from `shared.backend.utils.platform_config`, but the underlying JSON was missing — production deploys would log a "config file not found" warning. The shared config now provides a sensible default; per-platform overrides via `BIRD_PLATFORM_DATA_DIR` still work for branding. |
| Added `Production runtime contract (no demo mode)` step to `scripts/release_gate.ps1` | `scripts/release_gate.ps1` | New gate step that stages the production env vars, verifies all 4 `*_externalized` flags are `True`, validates the shared `platform_config.json` is loadable + has `platform.name`. Fails the gate if anyone tries to ship a build that would degrade to `demo` mode at runtime. |

## 6. Risk Register

| Risk | Likelihood | Mitigation |
|---|---|---|
| Production deploys forget to set the new env vars and the lifespan still hard-fails | Medium | The hard fail message explicitly names `SURVEY_DATA_DIR` and `BIRD_PLATFORM_DATA_DIR` so operators know what to set. Release gate enforces the contract at CI time. |
| Both platforms now load the same `shared/backend/utils/data/platform_config.json` so the acoustic platform shows field-survey branding | Medium | Per-platform override via `BIRD_PLATFORM_DATA_DIR=<acoustic_platform/backend/data>` still works. **Tracked as P1**: replace with a `PLATFORM_VARIANT` loader that picks `platform_config.{variant}.json` from the shared dir. |
| Staging snapshot `species_monitoring_platform/deploy/staging/backend/` does NOT receive these fixes | Low (it's a rollback artifact) | Will be removed or rebuilt during Batch B (code-twin elimination + Docker packaging). |
| Tests that import `main` and exercise the lifespan via `TestClient` may surface latent fail-fast errors that previously degraded to 503 | Low | Desired behaviour — fail-fast errors are easier to diagnose than chains of 503 responses. Tests that need a store can patch `main.survey_store = SimpleNamespace(...)` (the existing pattern in `test_health_runtime.py`). |
