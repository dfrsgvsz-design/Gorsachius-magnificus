# Decision Log 模板

> 路径：`submission/governance/decision_log.md`
> 任何 P0 或方向性决策必须留档，双签后生效。
> 失忆是项目最大成本——三个月后没人记得为什么这么做，就会拆掉重做。

## 单条决策格式

```markdown
## D-NN <一句话决议>

- 决策时间: 2026-MM-DD HH:MM
- 决策类型: <技术 / 产品 / 商业 / 合规 / 流程>
- 决策范围: <影响哪些模块、哪些里程碑>
- DRI: <谁来执行这个决策>
- 复评时间: <什么时候回头看这个决策对不对>

### 背景
<2–3 句话讲清楚为什么现在要决>

### 候选方案
| # | 方案 | 优点 | 缺点 | 工作量 | 风险 |
|---|---|---|---|---|---|
| A | ... | ... | ... | ... | ... |
| B | ... | ... | ... | ... | ... |
| C | ... | ... | ... | ... | ... |

### 选择
**方案 X**

### 选择依据
<3–5 句话，必须可追溯到事实/数据/约束，不允许"感觉"、"经验">

### 反方论证（必填——证明决策是经过质疑的）
- 反对方案 X 的最强理由是: <一句话>
- 我们为什么仍选 X: <一句话>

### 实施动作
- [ ] <动作 1> · DRI · 截止
- [ ] <动作 2> · DRI · 截止

### 监控触发条件（满足任一即重启决策）
- ...
- ...

### 双签
- PM: <名> · 2026-MM-DD
- 评审人: <角色 / 名> · 2026-MM-DD
```

## 哪些事必须入 Decision Log

| 类别 | 例子 |
|---|---|
| 技术选型 | 数据库换 PostgreSQL / 模型换 ConvNeXt / Map 库换 MapLibre |
| 架构边界 | 双 App 合并 / 拆 / 共享 shared/ |
| 产品取舍 | 删功能、加功能、改 onboarding |
| 商业 | 上架主体（个人 vs 机构）、定价、地区 |
| 合规 | 数据出境策略、删号路径、儿童条款 |
| 流程 | 改 release_gate 项、改闸门频次 |

## 哪些事不需要入 Decision Log

- 单个 bug 的修复方法
- 局部重构（除非影响 API 契约）
- 代码风格选择（应在 lint 配置里固化）
- 日常运维操作

## 复评机制

每个 Decision 都有"复评时间"，到时 PM 必须主动回看：

```
[复评] D-NN 在 2026-MM-DD 复评:
- 监控触发条件是否触发: Y/N
- 决策当时假设是否仍成立: Y/N
- 是否需要重新决策: Y/N
- 如重新决策，开新 D-NN+1，本条标记 SUPERSEDED
```

## 项目启动 Day 1 应入的 4 条决策（举例）

```markdown
## D-01 双 App 暂不合并，但共享 shared/ 必须使用
- 类型: 架构
- 选择: 维持双 App 上架，但 shared/backend 强制消费
- 依据: 合并工作量 ≥3 周 + 已有 keystore；共享 shared/ 当前重复度 60%
- 反方: 双 App 被审核员问"区别"风险中
- 复评: W4 末

## D-02 W1 内 Apple 账号 + Mac 资源是否到位 = iOS 上不上分水岭
- 类型: 商业
- 选择: 等 W1 末 sponsor 决断
- 依据: Apple 注册 3 周 vs Android 4 周路线
- 反方: 等待意味着 iOS W4 出不来
- 复评: W1 周五 16:00 闸门

## D-03 后端 W2 末必须切生产模式
- 类型: 技术
- 选择: A 工号在 W2 内完成 platform_config.json + 环境变量化
- 依据: backend_run.log 显示当前 demo 模式
- 反方: 切换可能引入新 bug
- 复评: W2 闸门

## D-04 启用 Play App Signing
- 类型: 合规
- 选择: 首发即启用，本地 keystore 降级为 upload key
- 依据: keystore 丢失即主体丢失
- 反方: 增加首次配置 1 天
- 复评: W3 闸门前
```

## 红线

- **没双签的决策不算决策** —— 任何人不得据此执行
- **没"反方论证"的草稿打回重写** —— 不能只有支持理由
- **复评时间不到 ≠ 不能重启** —— 任何 RAID 触发都可强制重启
- **决策记录公开** —— 透明化能挡住"会后又改"
