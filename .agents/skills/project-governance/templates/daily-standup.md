# L1 日报模板（每日 09:30 前提交）

> 路径：`submission/reports/daily/<YYYY-MM-DD>_<工号>.md`
> 时长：写作 ≤5 分钟。超 5 分钟说明你今天不知道在做什么，先和 PM 一对一。

```markdown
[日期] 2026-MM-DD
[工号] A / B / C / D / E
[角色] <一句话角色，可省略复用>

[昨日完成]
- 任务#: 把 X 做到 Y 状态（commit hash / PR # / 装机包名 / 截图链接）
- 任务#: ...

[今日计划]
- 任务#: 把 X 推到 Y 状态（明确出口标准）
- 任务#: ...

[障碍 / 依赖]
- 需要 <工号>: 提供 <具体物件> · 走 <什么路径>
- 等待 <人/系统>: 自 <什么时候> 起卡住

[红黄绿]
- 今日状态: 🟢绿 / 🟡黄 / 🔴红
  - 🟡/🔴 必须附一句原因 + 已采取的 1 步措施
```

## 工号差异化补充项（必填）

### A（后端）
- 上一轮 `release_gate.ps1` 在哪台机器跑的、结果是什么
- 生产 `/api/health` 24h 最低成功率

### B（前端/移动端）
- 今日是否出新调试 APK；若出，提供安装后冷启动截图
- 三大核心（地图/轨迹/记录）当前在哪个 commit 工作

### C（QA & DevOps）
- 主干今日绿色 commit 数 / 总 commit 数
- 仍未关闭的 P0 bug 数

### D（算法 & 数据）
- species mapping ↔ checkpoint head ↔ taxonomy manifest 三者一致性 Y/N
- 上一次推理 Top-1 准确率 vs 上一日 ±?

### E（UI/运营/法务联络）
- `submission/03_missing_required_business_inputs.md` 今日勾选变化
- 商店素材包当前完整度 %

## PM 处理日报的动作

1. 09:30–10:00 PM 扫一遍 5 份日报。
2. 任何 🔴：12:00 前发起 L+ 升级。
3. 任何"等待"超 48h：PM 直接联系阻塞方。
4. 累计 2 日未交：标红，转入个人发展记录。
