# 业务方一页纸周报模板

> 路径：`submission/reports/weekly/W<n>_business_oneview.md`
> 每周五 18:00 前由 PM 发出给 sponsor / 业务方 / 老板。
> **硬限制：一屏可读完。超过一屏 sponsor 不会看。**

## 模板（直接复制填充）

```markdown
# 海南鳽项目周报 W<n>（2026-MM-DD → 2026-MM-DD）

## 1️⃣ 进度（词不加形容词）

- **工程**: 上周 X% → 本周 Y%
- **业务**: <已勾选>/<总数> 项
- **预计 W4 上线概率**: ±??%（比上周 ↑/↓ N pp）

## 2️⃣ 三大风险（只报 3 条最大的）

| # | 风险 | 影响 | DRI | 缓解动作 |
|---|---|---|---|---|
| 1 | ... | ... | ... | ... |
| 2 | ... | ... | ... | ... |
| 3 | ... | ... | ... | ... |

## 3️⃣ 本周决策记录

- **[D-NN]** <一句话决议>
- **[D-NN]** <一句话决议>

## 4️⃣ 需 sponsor 本周拍板的 1–3 件事（含 deadline）

- [ ] **<事项>** —— 决策类型: <二选一/三选一> —— 必须在 <日期> 前回复
- [ ] ...

---

附：详细信息在 `submission/reports/weekly/W<n>_meeting_minutes.md`
```

## 写作纪律

| 维度 | 标准 |
|---|---|
| 总字数 | ≤ 600 字（含表格）|
| 形容词 | 0 个"显著"、0 个"大幅"、0 个"取得突破" |
| 数字 | 每段至少 1 个 |
| 时态 | 全部过去式 + 下周打算，禁止"正在" |
| 拍板请求 | 必须明确 deadline，不允许"尽快" |

## PM 该期待 sponsor 做什么

1. **48h 内回复"需拍板事项"** —— 超时 PM 直接按推荐方案执行，写入 Decision Log
2. **不在周报上当面修改进度数字** —— 数字是 PM/团队的；sponsor 修改 = 失信号
3. **重大决策走 Decision Log 双签** —— 口头变更不算数

## 与日报/周会的关系

```
日报 (L1) ──┐
            ├──► 周会 (L2) ──► 周会纪要 ──┐
1on1  (L3) ─┘                              ├──► 业务方一页纸 (本模板)
RAID Log 更新 ─────────────────────────────┤
闸门决策 (L4) ─────────────────────────────┘
```

业务方一页纸只承载 sponsor 必须知道的 4 类信息，其余细节通过链接给。
